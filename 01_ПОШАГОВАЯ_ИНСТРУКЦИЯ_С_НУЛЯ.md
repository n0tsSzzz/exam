## 1. Что делаем

Мы делаем учебный шаблон Django-проекта, который можно быстро переделать под экзаменационное задание.

В нашем тренировочном варианте предметная область похожа на магазин обуви:

- пользователи;
- роли;
- товары;
- поставщики;
- производители;
- категории;
- пункты выдачи;
- заказы;
- состав заказов;
- импорт данных из файлов;
- работа с фотографиями товаров.

Главная идея: на экзамене не переписывать проект с нуля в панике, а понимать порядок действий.

## 2. Проверка программ на компьютере

### Шаг 2.1 — Проверить Python

Открой PowerShell:

```powershell
python --version
```

или:

```powershell
py --version
```

Если команда показывает версию Python, значит Python установлен.

### Шаг 2.2 — Проверить PostgreSQL

```powershell
Get-Service -Name postgresql*
```

Нормально, если увидишь службу со статусом `Running`.

Если служба есть, но не запущена:

```powershell
Start-Service -Name postgresql*
```

### Шаг 2.3 — Проверить `psql`

```powershell
psql --version
```

Если команда не найдена, используй полный путь:

```powershell
& "C:\Program Files\PostgreSQL\16\bin\psql.exe" --version
```

> **Важно:** версия PostgreSQL может быть другой. Если стоит PostgreSQL 15, путь будет похож на `C:\Program Files\PostgreSQL\15\bin\psql.exe`.

### Шаг 2.4 — Проверить Git

```powershell
git --version
```

### Шаг 2.5 — Проверить UV

```powershell
uv --version
```

Если `uv` не найден, это не критично. Проект можно делать обычным способом через `.venv`.

## 4. Виртуальное окружение: обычный способ и UV

На экзамене используй тот способ, который быстрее работает на конкретном компьютере.

### Вариант А — обычный `.venv`

Создать окружение:

```powershell
py -m venv .venv
```

Активировать:

```powershell
.\.venv\Scripts\activate
```

Если PowerShell пишет, что выполнение сценариев отключено, запусти:

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
```

Потом снова:

```powershell
.\.venv\Scripts\activate
```

Установить зависимости:

```powershell
pip install django psycopg2-binary pillow
```

Сохранить зависимости:

```powershell
pip freeze > requirements.txt
```

### Вариант Б — UV

Если `uv` установлен:

```powershell
uv init --no-readme
uv venv
uv add django psycopg2-binary pillow
```

Команды Django можно запускать так:

```powershell
uv run python manage.py check
uv run python manage.py runserver
```

> **Важно:** UV все равно создает виртуальное окружение `.venv`, но обычно тебе не нужно вручную его активировать.

## 5. PostgreSQL: проверка и создание базы

### Шаг 5.1 — Подключиться к PostgreSQL

Обычный вариант:

```powershell
psql -U postgres
```

Если `psql` не найден:

```powershell
& "C:\Program Files\PostgreSQL\16\bin\psql.exe" -U postgres
```

Пароль в нашем тренировочном проекте:

```text
123456
```

### Шаг 5.2 — Посмотреть список баз

Внутри `psql`:

```sql
\l
```

### Шаг 5.3 — Создать базу

```sql
CREATE DATABASE demoexam_26;
```

Выйти:

```sql
\q
```

### Что значит `CREATE DATABASE demoexam_26;`

Эта команда создает новую пустую базу данных в PostgreSQL.

`demoexam_26` — это имя базы. Django потом будет создавать внутри нее таблицы через миграции.

Не нужно писать:

```sql
CREATE DATABASE postgres;
```

Потому что `postgres` обычно уже существует как служебная база по умолчанию.

## 7. Настройка `settings.py`

Открой файл:

```text
config/settings.py
```

### Шаг 7.1 — Добавить приложение

В `INSTALLED_APPS` добавь:

```python
"core",
```

### Шаг 7.2 — Разрешить запуск на любом хосте

```python
ALLOWED_HOSTS = ["*"]
```

Для экзамена так проще: проект запустится и локально, и если адрес будет отличаться.

### Шаг 7.3 — Подключить PostgreSQL

```python
DATABASES = {
    "default": {
        "ENGINE": "django.db.backends.postgresql",
        "NAME": "demoexam_26",
        "USER": "postgres",
        "PASSWORD": "123456",
        "HOST": "localhost",
        "PORT": "5432",
    }
}
```

### Шаг 7.4 — Настроить язык и время

```python
LANGUAGE_CODE = "ru-ru"
TIME_ZONE = "Europe/Moscow"
USE_I18N = True
USE_TZ = True
```

### Шаг 7.5 — Настроить статику и медиа

```python
STATIC_URL = "static/"
STATICFILES_DIRS = [BASE_DIR / "static"]

MEDIA_URL = "/media/"
MEDIA_ROOT = BASE_DIR / "media"
```

### Шаг 7.6 — Указать свою модель пользователя

```python
AUTH_USER_MODEL = "core.User"
```

> **Важно:** `AUTH_USER_MODEL` нужно указать до первой миграции. Если сначала сделать миграции, а потом поменять пользователя, будет сложнее исправлять.

### Шаг 7.7 — Упростить пароли на экзамен

```python
AUTH_PASSWORD_VALIDATORS = []
```

Так можно быстро создавать пользователей с простыми паролями.

## 9. Модели и связи

Модели пишутся в:

```text
core/models.py
```

### Основные таблицы

| Модель | Для чего нужна |
|---|---|
| `Role` | роли пользователей |
| `User` | пользователь системы |
| `Supplier` | поставщик товара |
| `Category` | категория товара |
| `Manufacturer` | производитель |
| `PickupPoint` | пункт выдачи |
| `Product` | товар |
| `Order` | заказ |
| `OrderItem` | позиция заказа |

### Связи

| Связь | Тип |
|---|---|
| `User -> Role` | многие к одному |
| `Product -> Supplier` | многие к одному |
| `Product -> Category` | многие к одному |
| `Product -> Manufacturer` | многие к одному |
| `Order -> PickupPoint` | многие к одному |
| `OrderItem -> Order` | многие к одному |
| `OrderItem -> Product` | многие к одному |

### Важные правила

- Для справочников используй `ForeignKey`.
- Для товара в заказе используй `on_delete=PROTECT`.
- Для состава заказа используй отдельную таблицу `OrderItem`.
- Для фотографии товара используй `ImageField`.
- В БД хранится путь к фото, а сам файл лежит в `media/products`.
- При замене фото старый файл нужно удалить.
- При удалении товара фото тоже нужно удалить.

## 11. Пользователи для входа

Админку Django в шаблоне не используем, потому что ее нет в требованиях экзамена.

Пользователи создаются через CSV-импорт:

```powershell

```

```text
import_data/user_import.csv
```

- через страницу входа проекта;
- через DBeaver в таблице `core_user`.

## 12. Импорт CSV

Импорт делается через management-команду:

```text
core/management/commands/import_data.py
```

### Что должна делать команда

- читать `.csv`;
- если данные выдали в Excel, сначала сохранить их как `CSV UTF-8`;
- понимать `utf-8-sig` и `cp1251` для CSV;
- пропускать пустые строки;
- создавать роли;
- создавать пользователей;
- создавать пункты выдачи;
- создавать справочники товаров;
- создавать товары;
- создавать заказы;
- создавать позиции заказов;
- не создавать дубли при повторном запуске.

### Какие файлы поддерживаем

| Данные | Файлы |
|---|---|
| Пользователи | `users.csv`, `user_import.csv` |
| Пункты выдачи | `pickup_points.csv`, `pp.csv`, `Пункты выдачи_import.csv` |
| Товары | `products.csv`, `tovar.csv`, `Tovar.csv` |
| Заказы | `orders.csv`, `order_import.csv`, `Заказ_import.csv` |

### Запуск импорта

```powershell
python manage.py import_data
```

Если данные лежат в другой папке:

```powershell
python manage.py import_data --path "C:\путь\к\папке"
```

После успешного импорта проверь в DBeaver, что появились товары, пользователи, заказы и пункты выдачи.

## 13. Авторизация и роли

### Роли

| Роль | Что может |
|---|---|
| Гость | смотреть товары без поиска, фильтра и сортировки |
| Клиент | смотреть товары без поиска, фильтра и сортировки |
| Менеджер | смотреть товары с поиском, фильтром, сортировкой; смотреть заказы |
| Администратор | полный CRUD товаров и заказов |

### Что должно быть на входе

- форма логина;
- форма пароля;
- кнопка входа;
- кнопка входа как гость;
- сообщения об ошибках;
- после входа справа сверху показывается ФИО пользователя.

### Выход

Выход лучше делать через POST-форму:

```django
<form method="post" action="{% url 'logout' %}">
    {% csrf_token %}
    <button type="submit">Выйти</button>
</form>
```

Так закрывается требование по CSRF для действия выхода.

## 14. Товары: список, поиск, фильтр, сортировка

### Что видят все роли

В списке товара должны быть:

- фото товара или заглушка;
- название;
- категория;
- описание;
- производитель;
- поставщик;
- цена;
- единица измерения;
- количество;
- скидка.

### Подсветка

| Условие | Что сделать |
|---|---|
| Скидка больше `15%` | фон строки `#2E8B57` |
| Есть скидка | старая цена красная и перечеркнутая |
| Есть скидка | новая цена черная |
| Количество `0` | выделить строку голубым |

### Поиск, фильтр и сортировка

Поиск, фильтр и сортировка доступны только менеджеру и администратору.

Требования:

- поиск по текстовым полям товара;
- фильтр по поставщику;
- первый пункт фильтра: `Все поставщики`;
- сортировка по количеству по возрастанию;
- сортировка по количеству по убыванию;
- поиск и фильтр работают вместе;
- выбранная сортировка сохраняется;
- изменения происходят без отдельной кнопки поиска.

## 15. CRUD товаров и работа с фото

CRUD товаров доступен только администратору.

### Добавление товара

На форме должны быть поля:

- фото;
- наименование;
- категория;
- описание;
- производитель;
- поставщик;
- цена;
- единица измерения;
- количество;
- скидка.

### Редактирование товара

Требования:

- форма открывается из списка товаров;
- поля заполняются данными из БД;
- если ID показан, он только для чтения;
- старое фото удаляется при замене;
- после сохранения список товаров обновляется.

### Удаление товара

Требования:

- удалять может только администратор;
- перед удалением должно быть подтверждение;
- товар нельзя удалить, если он есть в заказе;
- при удалении товара удаляется его фото;
- после удаления список обновляется.

### Pillow

Pillow нужен для изображений:

- проверить, что файл действительно является картинкой;
- изменить размер фото до `300x200`;
- сохранить оптимизированное изображение.

## 17. CSRF-защита

Django включает CSRF через middleware:

```python
"django.middleware.csrf.CsrfViewMiddleware",
```

Проверь, что эта строка есть в `MIDDLEWARE`.

Во всех POST-формах должен быть токен:

```django
{% csrf_token %}
```

Проверь формы:

- вход;
- выход;
- добавление товара;
- редактирование товара;
- удаление товара;
- добавление заказа;
- редактирование заказа;
- удаление заказа.

## 18. Удаление неиспользуемых фото

Команда:

```text
core/management/commands/cleanup_media.py
```

- не удаляет `.gitkeep`, если он есть.

Запуск:

```powershell
python manage.py cleanup_media
```
